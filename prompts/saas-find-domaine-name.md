## What Makes a Good Domain Name

A domain name should be:

- **Memorable**: Easy to remember after hearing once
- **Pronounceable**: Can be said clearly over phone/podcast
- **Spellable**: No ambiguous or weird spellings
- **Brandable**: Evokes the right feeling and associations
- **Available**: Actually available to register (critical!)
- **SEO-Friendly** (for utility names): Contains relevant keywords

The goal is to find an available domain that supports your brand strategy and product positioning.

## Critical Naming Context

**Research shows:**

- .com domains still have highest trust and recall (70% prefer .com)
- Developer audiences accept .dev, .io, .app equally
- Short names (1-2 words, 5-12 chars) perform best
- Invented words can be more brandable than real words (Figma, Stripe)
- Utility names provide SEO advantage (PageSpeed, WebChecker)
- Premium domains ($1000+) rarely worth it for early-stage

Your domain name is your first brand impression - invest time to get it right.

### Brand Domain Generation (Single Word)

**Approach**: Create unique, memorable, pronounceable names

**Techniques:**

1. **Invented Words**: Combine syllables (Figma = Fig + Figment, Vercel = Verifiable + Accelerate)
2. **Modified Real Words**: Twist existing words (Slack, Discord, Stripe)
3. **Compound Portmanteaus**: Merge two concepts (Instagram = Instant + Telegram)
4. **Abstract Concepts**: Evocative words that suggest the benefit (Notion, Linear, Aura)
5. **Short Real Words**: Simple, memorable words (Bolt, Frame, Wave)

**Quality Criteria:**

- ✅ Easy to pronounce (pass the "phone test")
- ✅ Easy to spell (no weird spellings like "Flikr")
- ✅ Memorable and distinctive
- ✅ No negative connotations in English
- ✅ Suggests the category or benefit
- ✅ 2-3 syllables ideal (5-10 characters)
- ✅ Sounds professional for target audience
- ❌ Avoid: Generic words, hard to spell, negative associations

**Word Banks by Category:**

_Monitoring/Tracking:_

- Vigilant, Sentinel, Beacon, Pulse, Radar, Scout, Watch, Guard

_Speed/Performance:_

- Rapid, Swift, Velocity, Instant, Bolt, Flash, Zoom, Dash

_Simplicity:_

_Intelligence/AI:_

- Sage, Cortex, Neural, Cognito, Mind, Brain, Think, Solve

_Building/Creation:_

- Forge, Craft, Build, Make, Studio, Workshop, Lab

_Data/Analysis:_

- Nexus, Matrix, Prism, Spectrum, Insight, Vision, Clarity

**Examples:**

- Monitoring tool: Vigilant, Sentinel, Beacon, Pulse, Watchtower
- Fast service: Rapid, Swift, Velocity, Instant, Bolt
- Simple tool: Plain, Clear, Stark, Pure
- AI product: Sage, Cortex, Neural, Prism

### Utility Domain Generation (Descriptive)

**Approach**: Clear, action-oriented, self-explanatory names that describe what the product does

**Patterns:**

## Domain Availability Validation

**CRITICAL**: You must check domain availability using actual domain lookup tools.

### Primary Method: WHOIS Command

Use the `whois` command via Bash tool to check availability:

```bash
whois example.com | grep -E "No match|NOT FOUND|No entries|Status: free|Available"
```

**Interpreting Results:**

- "No match" / "NOT FOUND" → ✅ Available
- "Status: free" → ✅ Available
- Shows registrar info → ❌ Taken
- "premium" / "reserved" → ⚠️ Premium pricing

### Alternative TLD Checking

If .com is taken, check alternative TLDs in priority order:

**For B2C SaaS:**

```bash
whois example.io | grep -E "No match|NOT FOUND"
whois example.app | grep -E "No match|NOT FOUND"
```

**For Developer Tools:**

```bash
whois example.dev | grep -E "No match|NOT FOUND"
whois example.io | grep -E "No match|NOT FOUND"
```

### Checking Workflow

For each generated name, check in this order:

1. Check primary TLD (usually .com)
2. If taken, check secondary TLD (based on audience)
3. Note availability status clearly
4. If premium, note that (usually detectable in whois)

**Document in this format:**

- ✅ **Available**: Can be registered immediately
- ❌ **Taken**: Already registered by someone else
- ⚠️ **Premium**: Available but at premium price ($500-$50k+)
- ❓ **Unknown**: Unable to verify (rare)

## Domain Registration Tips

After finding available domains, provide quick guidance:

**Where to Register:**

## Expert Naming Principles

**From Great SaaS Brands:**

**Stripe**: Simple, memorable, one syllable. Suggests speed and modernity.

**Notion**: Abstract but evocative. Suggests ideas and thinking.

**Vercel**: Invented word (Verifiable + Accelerate). Modern, technical.

**Linear**: Simple real word. Suggests clarity and focus.

**Figma**: Invented portmanteau (Fig + Figment). Playful, memorable.

**Key Patterns:**

1. Short (1-2 syllables) beats long
2. Real words work if they're simple (Linear, Stripe)
3. Invented words work if they're pronounceable (Figma, Vercel)
4. Abstract concepts can be powerful (Notion, Aura)
5. Utility names trade memorability for clarity (SendGrid, Mailchimp)
